GnuPG

Configuraciones para ambos sistemas operativos.

  • En Windows: %APPDATA%/gnupg/*
    • Instalar: Gpg4Win
    • Configurar los archivos.
  • En Ubuntu: ~/.gnupg/*
    • Instalar los paquetes: gnupg pcscd pcsc-tools scdaemon
    • Configurar los archivos.
    • Reiniciar servicios: systemctl --user restart gpg-agent y sudo systemctl restart pcscd
    • Obtener la ruta del Socket: gpgconf --list-dirs agent-ssh-socket
    • Configurar la variable: SSH_AUTH_SOCK="/run/user/1000/gnupg/S.gpg-agent.ssh" en ~/.bashrc

Terminal Bash

Editar con: code ~/.bashrc y luego recargar con: source ~/.bashrc

Tambien se puede instalar Oh My Zsh! de la siguiente forma:

  1. Instalar sudo apt install zsh git
  2. Configurar la terminal para usar zsh: chsh -s $(which zsh)
  3. Ejecutar el comando que indican en la Web Oficial: 
    sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"
  4. Instalar los plugins: ZSH-AutoSuggestions y ZSH-Syntax-HighLighting: 
    git clone https://github.com/zsh-users/zsh-autosuggestions ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-autosuggestions
git clone https://github.com/zsh-users/zsh-syntax-highlighting.git ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-syntax-highlighting
  5. Habilitar los modulos en .zshrc colocando: plugins=(git zsh-autosuggestions zsh-syntax-highlighting).
  6. Instalar las fuentes para los iconos: FiraMono > Regular
  7. Instalar el tema PowerLevel10K. 
    git clone --depth=1 https://github.com/romkatv/powerlevel10k.git "${ZSH_CUSTOM:-$HOME}/.oh-my-zsh/custom/themes/powerlevel10k"
  8. Volver a editar el .zshrc y configurar: ZSH_THEME="powerlevel10k/powerlevel10k"
Dependiendo de la configuracion elegida, si se utilizo Oh My Zsh! se cargaran las funciones en .zshrc, en caso contrario, será en .bashrc.

Las siguientes configuraciones son para implementar la conexion SSH utilizando la funcionalidad PIV u OpenPGP del YubiKey en Windows. En caso de estar en Linux y querer usar OpenPGP, seguir las intrucciones de GnuPG



Instalación en Windows

Opcion 1: Yubico PIV Tool, o bien OpenSC

  1. Instalar Yubico PIV Tool, o bien OpenSC.
  2. (Opcional) Luego ubicar la carpeta donde se instaló y asignar ese path a las variables del sistema.
  3. Se deberá configurar el archivo .ssh\config ubicado en la raíz del usuario C:\Users\{nombre}. Allí colocar la configuración (según el software elegido): 
    PKCS11Provider "C:\Program Files\Yubico\Yubico PIV Tool\bin\libykcs11.dll"
PKCS11Provider "C:\Program Files\OpenSC Project\OpenSC\pkcs11\opensc-pkcs11.dll"
  4. Una vez configurado esto, probar la conexión ssh con algún servidor.


Opcion 2: OpenPGP

Se utiliza el software Gpg4Win para usar el gpg-agent como agente de autenticación de SSH.

Los pasos a seguir son sencillos, instalar: Gpg4Win y luego elegir la configuracion de los archivos correspondientes a Windows (OpenPGP)

  • Nota 1: Se debe colocar la linea: gpgconf --launch gpg-agent en el archivo $PROFILE de Windows.
  • Nota 2: La variable SSH_AUTH_SOCK no debe estar definida para que gpg-agent funcione correctamente.
    Para eliminarla, se utiliza: Remove-Item Env:\SSH_AUTH_SOCK

Algunos comandos útiles a la hora de trabajar con gpg-agent son:

  1. gpgconf --kill gpg-agent: Detiene el agente.
  2. gpgconf --launch gpg-agent: Inicia un agente (recarga las configuraciones también)
  3. ssh-add -L: Si todo funciona, muestra las claves SSH que se pueden usar (aquellas que lee del YubiKey)
  4. gpg-connect-agent "scd serialno" /bye: Consulta el numero de serie de la YubiKey. Esto prueba la comunicacion del agente con la misma.


Instalación en Linux (Ubuntu, Mint, ...)

Opcion 1: OpenSC

Simplemente ejecutar: sudo apt install opensc y luego verificar que exciste el archivo /usr/lib/x86_64-linux-gnu/opensc-pkcs11.so



Opcion 2: Yubico PIV Tool

Instrucciones recopiladas de Yubico PIV Tool:

  1. Instalar los siguientes paquetes 
    sudo apt install cmake libtool libssl-dev pkg-config check libpcsclite-dev gengetopt help2man zlib1g-dev
  2. Luego descargar la ultima version de: Yubico PIV Tools Releases
  3. Descomprimir y compilar: 
    cd yubico-piv-tool
mkdir build; cd build
cmake ..
make
sudo make install
  4. Si se logro instalar correctamente, mostrará un listado de las ubicaciones donde quedaron los archivos. Con esto se configura el archivo ~/.ssh/config para que utilice la YubiKey: 
    PKCS11Provider /usr/local/lib/libykcs11.so
  5. Se deberá recargar las librerías con: sudo ldconfig y finalmente probar la conexión ssh con algún servidor.
