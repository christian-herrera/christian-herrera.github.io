Introducción

Que es R?

R es un lenguaje de programación que combinado con RStudio (Software) nos permite explorar y analizar una amplia gama de fenómenos sociales de los cuales tengamos datos.
Utiliza librerías y módulos complementarios para ampliar sus funcionalidades (Quarto).

Que es Quarto?

Es un sistema de publicación científica y técnica de código abierta.
Permite crear contenido dinámico con Python, R, Julia y Observable.
Operativamente genera documentos de alta calidad, que suponen la ejecución de código, generación de texto con formato y gráficos.

Quarto admite una variedad de formatos para crear presentaciones, que incluyen:

  • revealjs: Link (HTML)
  • pptx: Link (MS Office)
  • beamer: Link (LaTeX/PDF)

De todos estos formatos, el mas recomendable es Reveal.js. Estas presentaciones podrán presentarse en formato HTML o en PDF para facilitar su distribución.

Crear Diapositivas

Estructura Principal
...

Cada diapositiva se crea como en obsidian (utilizando Markdown), solo que se limita a dos niveles: # y ##.

---
title: "Habits"
author: "John Doe"
format: revealjs
---
# Titulo 1
## Titulo 1.1

- Turn off alarm
- Get out of bed

## Titulo 1.2

- Get in bed
- Count sheep
Quitar la diapositiva de Titulo

Si se coloca las etiquetas title y author, automaticamente se creara una primer diapositiva con estos datos. En caso de ser necesario, se puede eliminar estas etiquetas y no existira esta primer diapositiva.

Sintaxis
...

Listas Incrementales
...

De forma predeterminada, las listas de números y viñetas dentro de las diapositivas se muestran todas a la vez. Puede anular esto globalmente o bien de forma mas puntual:

Para Todas las Listas
...

---
title: "Habits"
author: "John Doe"
format: 
	revealjs:
		incremental: true
---
# Titulo 1
## Titulo 1.1

De Forma Especifica
...

::: {.incremental}
- Eat spaghetti
- Drink wine
:::

Desbordamiento de Contenido
...

Si se tiene una diapositiva con mas contenido del que se puede mostrar en un solo cuadro, hay dos formas de solucionarlo:

Usar una Etiqueta
...

Esto modifica el tamaño de la letra:

## Titulo 1.1 {.smaller}

Hacer la Scrollable la Diapositiva
...

## Titulo 1.1 {.scrollable}

Este valor se puede colocar en el codigo YAML para hacer scrollable todas las diapositivas:

---
format:
  revealjs:
    smaller: true
    scrollable: true
---

Resaltado de Linea
...

Es posible que se desee resaltar algunas lineas de codigo. Para esto se utiliza el atributo code-line-numbers: Por ejemplo, en la imagen anterior, para resaltar desde la linea 2 a la linea 3, le debo agregar:

{.python code-line-numbers="2-3"}
{.python code-line-numbers="2,3"}

Otra funcionalidad es hacerlo de manera incremental, es decir, que se valla resaltando de una en una al presionar la tecla.

{.python code-line-numbers="|2|3"}

Numero de Línea
...

Por defecto ya viene habilitado pero se puede quitar con el atributo:

code-line-numbers="0"

Transiciones
...

Reveal admite una serie de efectos de transicion animados tanto para cambios de diapositivas como cambios de fondo de diapositiva. Se configuran con los atributos YAML:

transition: slide
background-transition: fade
transition-speed: default    #[default], [fast] o [slow]

Donde los valores disponibles son:

TraniscionesDescripcion
noneSin traniscion
fadeDesvanecimiento cruzado
slideDeslizar horizontalmente
convexDeslizar en angulo convexo
concaveDeslizar en un angulo concavo
zoomEscale la diapositiva entrante para que crezca desde el centro de la pantalla.

QRCode
...

Esta extensión permite generar códigos QR en el mismo renderizado. Primero se deberá instalar la extensión por tal motivo, debo ubicarme con una terminal de PowerShell (o la terminal de linux) en la carpeta donde se encuentra el archivo de presentación. Luego ejecutar:

quarto install extension jmbuhr/quarto-qrcode

Esto descargara y configurara la extensión únicamente para el proyecto en cuestión.

Luego, en la presentación, se puede dejar una diapositiva que muestre el código QR brindando la web para que los clientes se conecten, una opción podría ser:

::: {style="text-align: center"}
# Titulo {background-iframe="assets/background/index.html"}
{{< qrcode https://christian-herrera.gitlab.io/repositorio colorDark='#000000' width=200 height=200 >}}
[christian-herrera.gitlab.io/repositorio](https://christian-herrera.gitlab.io/repositorio/)
:::

Se deberán cambiar las url para que funcione correctamente.

Recordar que el código QR se logra centrar con los valores colocados en el CSS aquí

Personalizar Extensión
...

He encontrado que al generar códigos QR en plantillas que tienen el fondo oscuro, no se logra reconocer bien por los dispositivos, así que decidí buscar la línea de codigo del archivo JavaScript que modifica el CSS al momento de generar la imagen.

  1. Buscar el archivo: \_extensions\jmbuhr\qrcode\assets\qrcode.js
  2. Encontrar la linea: this._elImage.style.display = "block";
  3. Escribir debajo: this._elImage.style.border = "6px solid white";

Con esto se logra un borde mas ancho (de 6 px) y permite tener el cambio de negro a blanco en los bordes.

Para generar un código en RStudio, utilizar:

Multiplex Server
...

Multiplex server es un sistema que permite compartir la presentación con la ventaja de que cada usuario dispondrá en vivo de la presentación (actualizando la diapositiva a la que el hablante esta presentando).

Para configurar esto, se tienen dos métodos, utilizando una Servidor Web privado, o bien usando GitLab o GitHub.

Donde esta ubicado el servidor Socket.IO?

Para que esto funcione, existe un servidor Reveal Multiplex corriendo en todo momento, y al momento de hacer un render, se crea otro servidor pero local.
Entonces, tanto los clientes como la maquina del presentador, harán uso de ese servidor, de tal forma que se logran transmitir eventos provenientes del master, hacia los clientes.

Utilizando un Servidor Privado
...

Configuración en la presentación
...

---
format:
  revealjs:
    multiplex: true
---

Ahora, se deberá renderizar la presentación, haciendo esto obtienen dos archivos html:

  • Presentacion.html: El archivo que se debe compartir.
  • Presentacion-speaker.html: El archivo local, que presentara el orador.

y además el resto de los recursos. De todos ellos, deben copiarse a la carpeta /presentacion del servidor web, los siguientes archivos:

  • Carpeta *_files
  • Archivo Presentacion.html
  • Resto de archivos HTML, JavaScript, CSS utilizados.

Estos archivos se deberán copiar en la ruta del servidor. Una vez realizado, cambiar el nombre de Presentacion.html a index.html.

Establecer Permisos
...

Cada archivo deberá tener los permisos: 644, mientras que cada directorio deberá tener 644.
Proceder colocando a todo, los permisos 644:

chmod -R 644 *

Y luego, a cada directorio colocarle 744:

find ./ -type d -exec chmod 755 {} +

Ejecucion
...

Cada usuario podrá conectarse a la presentación utilizando el link hacia el archivo index.html que se alojo en el servidor. Cuando el navegador del cliente ejecute este HTML y JavaScript, se conectara al servidor de Reveal Multiplex y funcionara en conjunto con el master.

GitLab

Crear el Repositorio
...

Primero se deberá crear un repositorio vacío. Luego, colocar junto al README.md, todos los archivos nombrados en la sección anterior.
Adicionalmente, se deberá crear un archivo llamado: .gitlab-ci.yml que deberá contener:

pages:
  stage: deploy
  script:
    - mkdir .public
    - cp -r * .public
    - rm .public/Backup.zip.asc
    - mv .public public
  artifacts:
    paths:
      - public
  only:
    - main

La descripción de cada línea se puede resumir en la siguiente tabla:

LíneaDescripción
stage: deployIndica que estas instrucciones forman parte de la etapa de despliegue dentro de un pipeline de CI/CD. La etapa se llama deploy.
script:Aquí se define una lista de comandos que se ejecutarán en orden como parte de la etapa de despliegue. En este caso, crea una carpeta temporal .public, copia todo alli y luego la renombra a public.
artifacts:Define qué archivos o directorios se deben guardar como artefactos de esta etapa del pipeline.
paths: publicEspecifica que se mantendrá la carpeta public después de se ejecute el pipeline.
onlyDefine las condiciones bajo las cuales se ejecutara esta configuración, en este caso será solo para la rama principal main.
En resumen...

Este archivo YAML describe un flujo de trabajo de CI/CD que realiza tareas como la creación de un directorio oculto, la copia de archivos y directorios a ese directorio, el renombramiento de este directorio y la especificación de un directorio (public) como artefacto para su uso posterior. Todo esto ocurre solo cuando se realiza un push o merge a la rama principal del repositorio.

Ejecución
...

Esto se realiza únicamente en la computadora del presentador. El objetivo es que brinde un servidor multiplex al cual el navegador pueda usar.

En Ubuntu (Usando static)
...

  1. Primero se debera instalar: node-static (utilizando apt)
  2. Luego, en la carpeta del proyecto, renombrar el archivo xxxxxxxx-speaker.html a index.html.
  3. Finalmente, ejecutar: static . localhost:9090 (Debo estar en la misma carpeta que el index.html).

En Ubuntu (Usando node)
...

  1. Primero se deberá instalar nodejs y npm, para esto usar: sudo apt install nodejs npm

  2. Luego, en la carpeta del proyecto, renombrar el archivo xxxxxxxx-speaker.html a index.html.

  3. Copiar el archivo index.html y el resto de recursos a una carpeta nueva llamada /public

  4. En esa carpeta, crear un archivo server.js y colocarle dentro:

    /**
 * Static HTTP Server
 *
 * Create a static file server instance to serve files
 * and folder in the './public' folder
 */

// modules
var static = require( 'node-static' ),
    port = 8080,
    http = require( 'http' );

// config
var file = new static.Server( './public', {
    cache: 3600,
    gzip: true
} );

// serve
http.createServer( function ( request, response ) {
    request.addListener( 'end', function () {
        file.serve( request, response );
    } ).resume();
} ).listen( port );

  5. Finalmente, ejecutar el servidor con node server.js

En Windows (Usando Powershell)
...

  1. Instalar NodeJs, con winget install OpenJS.NodeJS
  2. Continuar con los pasos igual que en Ubuntu (Usando NodeJs)
