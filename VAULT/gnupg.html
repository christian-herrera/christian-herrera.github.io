Gestión de Claves
...

Un par de claves se compone de una clave publica y otra privada. Pero a su vez, estas partes tienen varios detalles que son interesantes de conocer:

  • Clave Publica Maestra
    • Parte Publica de la Clave de Firmado Maestra
    • Partes Publicas de las Sub-Claves de Firmado y Cifrado
    • Identificadores de Usuario (Se utiliza para asociar la clave publica con una persona real)
  • Clave Privada Maestra
    • Partes Privadas de las Claves

Ejemplo:

pub   rsa4096 2023-01-08 [SC]
      87F44023A98DE42AF19274E8AD175429B6768C5C
uid        [  absoluta ] Christian Yoel Herrera <___EMAIL___>
sub   rsa4096 2023-01-08 [E]
sub   rsa4096 2023-01-08 [S]
sub   rsa4096 2023-01-08 [A]

Creando el Par de Llaves GPG Perfecto
...

Este punto, esta resumido del blog de Alex Cabal.
La estructura es la siguiente:

  • Clave Publica y Privada (Maestra)
    • Subclave de Encriptación [E]
    • Subclave de Firma [S]

El par (Maestro) se utilizara para crear las subclaves nomas, luego de esto sera eliminado de la pc para almacenarlo en un USB o dispositivo offline.
Cada clave de Encriptación-Firma será utilizado en un Yubikey o en la maquina de uso diario, la idea es que si esta clave se ve comprometida, automáticamente se podrá revocar y se generara un nuevo par [ES] reemplazando la que esta en uso.

Genero el Par Maestro
...

  • gpg --expert --full-gen-key
    • Elegir RSA de 4096 bits
    • Colocar una fecha a elección
    • Colocar Nombre, Apellido y Email.

Agrego el par de Claves para Encriptacion y Firma
...

  • gpg --K --keyid-format long
  • gpg --edit-key [ID master]
    • con addkey crear una clave RSA de 4096 solo para Firma
    • con addkey crear una clave RSA de 4096 solo para Encriptacion

Genero el Certificado de Revocacion
...

  • gpg --armor --output cert_revocacion.asc --gen-revoke [ID master]

Exporto todas las Claves
...

  • gpg --armor --output public.asc --export [ID master]
  • gpg --armor --output private.asc --export-secret-key [ID master]
  • gpg --armor --output subkeys.asc --export-secret-subkeys [ID master]
  • gpg --armor --output subkey.E.asc --export-secret-subkeys [ID subclave de encriptacion]!
  • gpg --armor --output subkey.S.asc --export-secret-subkeys [ID subclave de firma]!

Elimino todas las claves privadas
...

  • gpg --delete-secret-key [ID master]
  • gpg --delete-secret-key [ID subclave de encriptacion]!
  • gpg --delete-secret-key [ID subclave de Firma]!

Importo a la PC solo las SubClaves
...

  • gpg --import subkey.E.asc subkey.S.asc

Generando un Par de Llaves para SSH
...

Para este caso, se utiliza el comando ssh-keygen. Este mismo genera una clave privada y una clave publica en el formato que espera los sistemas linux.
Algunos parametros son:

ParametrosDescripcion
-tEspecifica el tipo de clave: rsa, dsa, ecdsa y ed25519
-bDefine el tamaño en bits, por ejemplo: 2048, 4096.
-CPermite agregar un comentario a la clave.
-fEspecifica el nombre del archivo que se generara.
-NPermite establecer una contraseña a la clave privada.
-qHace que el comando funcione en modo silencioso
-yGenera una clave publica a partir de una clave privada.
-lMuestra la longitud en bits de una clave publica.
-pPermite cambiar la contraseña de una clave privada.

Ejemplo

ssh-add -t rsa -b 4096 -C "Server LP"

Utilizando un Servidor de Claves
...

Para este ejemplo, el servidor es keys.openpgp.org. La clave ya esta almacenada en este servidor por lo que solamente sera necesario hacer la importación.

  • gpg --keyserver keys.openpgp.org --recv-keys 87F44023A98DE42AF19274E8AD175429B6768C5C

Exportación
...

El parámetro --armor es para que el archivo sea legible, y que no este en formato binario.

Clave Publica
...

Primero visualizar el listado completo de las claves almacenadas: gpg --list-keys --keyid-format=long
Luego, exportarla con:

  • gpg --armor --output [clave_pub.gpg] --export [ID_de_la_clave_pública]
Subir a un Servidor de Claves

Se puede subir directamente con el comando gpg --keyserver keys.opengpg.org --send-key [ID del anillo]

Clave Privada
...

Se visualiza de la misma forma. Luego se exporta con:

  • gpg --armor --output [clave_priv.gpg] --export-secret-key [ID_de_la_clave_privada]

Subclave Privada
...

Se logra exportar la clave privada de una unica subclave o de todas las subclaves del anillo:

Todas las Subclaves

  • gpg --armor --output subkeys.asc --export-secret-subkeys [ID del anillo]
  • gpg -a -o subkeys.asc --export-secret-subkeys [ID del anillo]

Solo una unica Subclave
Asegurarse de colocar el ! al final del ID ya que esto permite exportar esa unica subclave y no todas.

  • gpg --armor --output subkey.enc.asc --export-secret-subkeys [ID de la subclave]!
  • gpg -a -o subkey.enc.asc --export-secret-subkeys [ID de la subclave]!

Cifrado Simétrico
...

Es el tipo de cifrado mas sencillo, mas rápido de procesar pero también mas inseguro.
Se puede colocar el parametro --output [nombre.gpg] para dejar el archivo cifrado con un nombre especifico.

Cifrar un Archivo
...

  • gpg -c Texto.txt
  • gpg --symmetric Texto.txt
    Solicitara una contraseña. Después de ingresarla se generara un archivo con la extensión .gpg.

Descifrado de un Archivo
...

Para el ejemplo, utilizare el mismo archivo del punto anterior.

  • gpg -d Texto.txt.gpg
  • gpg --decrypt Texto.txt.gpg
    Solicitara la misma clave del paso anterior.

Cifrado Asimetrico
...

El metodo consiste en cifrar un archivo con la clave publica y que solo el dueño de la clave privada puede descifrar el archivo y obtenerlos.

Cifrar Archivo
...

Si el archivo se llama Texto.txt, entonces las opciones son:
Agrega .sig al nombre original

  • gpg --recipient [ID del destinatario] --encrypt Texto.txt
  • gpg -r [ID del destinatario] --e Texto.txt

Nombre de Salida personalizado

  • gpg --output [nombre.gpg] --recipient [ID del destinatario] --encrypt Texto.txt
  • gpg -o [nombre.gpg] -r [ID del destinatario] -e Texto.txt

NOTA: El ID del destinatario sera la clave publica del mismo, al cifrarse con esta clave no solicitara contraseña pero la misma deberá estar agregada en la maquina antes de cifrar.

NOTA: El archivo se comprime antes de cifrarse.

Descifrar Archivo
...

Si el archivo se llama Texto.txt.gpg, entonces las opciones son:

  • gpg --output Texto.txt --decrypt Texto.txt.gpg
  • gpg -o Texto.txt -d Texto.txt.gpg

NOTA: La clave privada del par Public-Private que se utilizo para cifrar debera estar instalada en la maquina antes de intentar descrifrar el archivo.

Firma Digital
...

Una firma digital certifica un documento y le añade una marca de tiempo. Si posteriormente el documento fuera modificado en cualquier modo, el intento de verificar la firma fallaría. La utilidad de una firma digital es la misma que la de una firma escrita a mano, sólo que la digital tiene una resistencia a la falsificación.

Para la creación y verificación de firmas, se utiliza el par público y privado de claves en una operación que es diferente a la de cifrado y descifrado. Se genera una firma con la clave privada del firmante. La firma se verifica por medio de la clave pública correspondiente.

Opción 1: Empaquetado de Archivo y Firma
...

Firma
...

La opción de línea de órdenes --sign se usa para generar una firma digital. El documento que se desea firmar es la entrada, y la salida es el documento firmado.

Sin Nombre Personalizado

  • gpg --sign Texto.txt
  • gpg -s Texto.txt

Con Nombre Personalizado

  • gpg --output Texto.txt.gpg --sign Texto.txt
  • gpg -o Texto.txt.gpg -s Texto.txt
Formato de Salida

Si se agrega el parámetro --armor o -a, lo unico que cambia es que la extensión será .asc. En caso contrario, será .gpg

Seleccionar Clave

Si se dispone en el equipo de mas de una llave, entonces se podrá seleccionar cual de todas firmara (o cifrara) el archivo, el parámetro correspondiente es --local-user [ID de la clave] o -u [ID de la clave].

Verificación y Recuperación
...

El archivo queda empaquetado junto a su firma, en este punto se tienen dos opciones:

Verificar la firma

  • gpg --verify Texto.txt.gpg

Verificar y Desempaquetar

  • gpg --decrypt Texto.txt.gpg
  • gpg -d Texto.txt.gpg

Opción 2: Empaquetado de Archivo y Firma ASCII
...

Las firmas digitales suelen usarse a menudo para firmar mensajes de correo electrónicos o en los grupos de noticias. En estas situaciones no se debe comprimir el documento al firmarlo, ya que para aquellos que no dispongan de un sistema para procesarlo sería ininteligible.

Firma
...

Para firmar se utiliza el parametro --clearsign.

  • gpg --clearsign Texto.txt
Formato de Salida

En esta opcion, como la idea es firmar mensajes legibles, entonces se adjuntara el texto y seguido la firma digital.

Seleccionar Clave

Si se dispone en el equipo de mas de una llave, entonces se podrá seleccionar cual de todas firmara (o cifrara) el archivo, el parámetro correspondiente es --local-user [ID de la clave] o -u [ID de la clave].

Verificación y Recuperación
...

El archivo queda empaquetado junto a su firma, en este punto se tienen dos opciones:

Verificar la firma

  • gpg --verify Texto.txt.gpg

Verificar y Desempaquetar

  • gpg --decrypt Texto.txt.gpg
  • gpg -d Texto.txt.gpg

Opción 2: Archivo y Firma por Separado
...

Un documento firmado tiene una utilidad limitada. Los otros usuarios deben recuperar la versión original del documento de la versión firmada, y aun en el caso de los documento firmados en ASCII, el documento firmado debe ser editado para poder recuperar el original. Por tanto, existe un tercer método para firmar un documento, que genera una firma acompañante. Para generar una firma acompañante se usa la opción --detach-sig.

Firma
...

Para firmar se utiliza el parámetro --detach-sign o -b.

Sin Nombre Personalizado

  • gpg --detach-sig Texto.txt
  • gpg -b Texto.txt

Con Nombre Personalizado

  • gpg --output Texto.txt.sig --detach-sig Texto.txt
  • gpg -o Texto.txt.sig -b Texto.txt
Formato de Salida

Si se agrega el parámetro --armor o -a, lo único que cambia es que la extensión será .asc. En caso contrario, será .gpg

Seleccionar Clave

Si se dispone en el equipo de mas de una llave, entonces se podrá seleccionar cual de todas firmara (o cifrara) el archivo, el parámetro correspondiente es --local-user [ID de la clave] o -u [ID de la clave].

Verificación
...

Para realizar la verificación, con el archivo que se firmo y la firma acompañante, ambos en la misma ubicación. Se utilizara el parámetro --verify para verificar.

  • gpg --verify Texto.txt.sig

Uso Diario
...

ComandoDescripcion
gpg --keyserver keys.openpgp.org --recv-key [ID]Trae de un servidor de Claves
gpg --keyserver keys.openpgp.org --send-key [ID]Sube a un servidor de Claves
gpg --keyserver keys.openpgp.org --refresh-keysActualiza de un servidor de Claves
--
gpg --symetric [Archivo]Cifrado Simetrico
gpg --decrypt [Archivo]Descifrado Simetrico
--
gpg --armor --encrypt --recipient [ID]{!} [Archivo]Cifrado Asimetrico
gpg --decrypt [Archivo]Descifraco Asimetrico
--
gpg --sign --local-user [ID]{!} [Archivo]Firma y Empaqueta
gpg --clear-sign --local-user [ID]{!} [Archivo]Firma y Empaqueta (ASCII)
gpg --detach-sig --local-user [ID]{!} [Archivo]Firma Acompañante
gpg --verify [Archivo]Verifica la firma sin Desempaquetar
gpg --decrypt [Archivo]Verifica la firma y Desempaqueta
Observaciones
  • Para el Cifrado Asimetrico, si utilizo una subclave, debo colocar el ! para forzarlo.
  • El cifrado utiliza una clave publica, por tanto el descifrado utilizara la clave privada correspondiente.
  • La firma utiliza la clave privada, por tanto la verificacion utilizara la clave publica correspondiente.
